# 🐾 포디(Pawdy) 프로젝트 개발 및 버그 수정 완료 리포트

본 문서는 지금까지 진행된 모든 주요 피처 개발, 비주얼 리브랜딩, API 통합 및 버그 수정 세부 내역을 요약하여 다른 AI 모델(예: Claude)이나 작업자가 이어서 고도화 작업을 진행할 수 있도록 돕는 연동 명세서입니다.

---

## 📌 1. 앱 로고 및 비주얼 리브랜딩

* **3D 젤리 발바닥 앱 아이콘 탑재**:
  * 기존의 스큐어모피즘 형태에서 입체적인 3D 젤리 질감을 강조한 세련된 모던 아이폰 화풍 아이콘으로 전격 리뉴얼했습니다.
  * 바탕화면에 설치 시 둥근 테두리 겹침 없이 꽉 차게 보이도록 **풀블리드(Full-Bleed) 아웃라인 가공`**을 완료했습니다.
* **상태바 알림(Push Notification) 아이콘 연동**:
  * 결제나 주문 상태 변경 시 상단 상태바에 뜨는 아이콘을 기본 Flutter 로고 대신 **3D 발바닥 형태의 포디 전용 마크**로 완전 대체했습니다.

---

## 📌 2. 토스페이먼츠 결제 시스템 안정화

* **결제 준비 실패 오류 해결**:
  * 토스페이먼츠 결제 준비창(시뮬레이터) 진입 시 상태 관리 동기화 타이밍 불일치로 발생하던 실패 연동을 해결하여, 실제 테스트 결제 완료 및 백엔드 영수증 대조까지 물 흐르듯 작동합니다.

## 📌 3. 주문 내역 기능 대대적 고도화

* **구매확정 기능 신설**:
  * 주문 상세 내역에 **`구매확정`** 버튼을 추가했습니다.
  * 클릭 시 확인 다이얼로그가 뜨고 상태가 `구매확정`으로 변경되며, 고객에게 **구매확정 푸시 알림**이 도달하도록 동기화했습니다.
* **리뷰 작성 분기 처리 (단일 vs 다중 상품)**:
  * **주문 상품이 1개인 경우**: `리뷰작성`을 누르면 즉시 사진 및 별점을 입력하는 리뷰 작성 바텀시트가 노출됩니다.
  * **주문 상품이 여러 개인 경우**: 어떤 옷에 대해 리뷰를 작성할지 고르는 **상품 선택기 바텀시트**가 먼저 팝업된 뒤, 선택한 의류에 매핑되는 리뷰 세션이 열리는 사용자 친화적 UI를 구축했습니다.

## 📌 4. 구글(Google) 로그인 API 키 및 네이티브 연동

* **7.x 최신 싱글톤 규격 탑재**:
  * Flutter 구글 로그인 공식 플러그인(`google_sign_in: 7.2.0`)의 아키텍처 개편에 대응하여 **싱글톤 호출 방식**(`GoogleSignIn.instance.initialize` 및 `authenticate`)으로 구현을 마쳤습니다.
* **백엔드(FastAPI) 실서빙 크립토 토큰 검증**:
  * 클라이언트로부터 전달받은 `idToken`을 구글 공식 `Tokeninfo API`와 직접 교환 및 서명 검증(Audience, 만료 기간, 서명자 등)하는 견고한 보안 코드를 연동했습니다.
* **하이브리드 자동 분기 (실서빙 vs 개발 시뮬레이터)**:
  * 백엔드 환경변수 `PETFIT_GOOGLE_CLIENT_ID`가 비어 있을 때는 모의 시뮬레이터 창이 뜨고, 값이 등록되면 실제 구글 계정 팝업이 뜨도록 자동 분기 처리했습니다.
* **구글 콘솔 인증 설정 규칙**:
  * 현재 구글 클라우드 콘솔의 OAuth 상태는 **`프로덕션 단계(게시됨)`**이므로 테스트 이메일을 수동으로 추가 등록할 필요가 없습니다.
  * **작동 조건**: 구글 콘솔에 **`웹 애플리케이션 키`**와 **`Android 클라이언트 키`**가 **동시에 존재**해야 합니다. (웹 키는 Railway `PETFIT_GOOGLE_CLIENT_ID` 변수에 할당되며, 안드로이드 키는 패키지명 `app.pawdy.pawdy`와 SHA-1 `BE:44:EB:9E:E4:AB:CA:3C:B3:A1:F0:9B:AB:BC:37:66:BA:BE:C7:1F`로 구성되어 폰 내의 구글 플레이 서비스 인증을 담당합니다.)

## 📌 5. 상점 관리자(판매자 대시보드) 상품 관리 고도화

* **상단 요약 통계 카드형 필터 신설**:
  * **`전체 상품`**, **`품절 임박 (1~5개)`**, **`품절 상품 (0개)`** 수량을 자동으로 집계하여 카드 형태로 보여줍니다.
  * 각 요약 카드를 터치하면 하단의 등록 상품 목록이 **해당 상태의 제품들로 실시간 필터링**됩니다.
* **실시간 라이브 검색창**:
  * 상품명 입력 시 글자 입력 타이밍에 맞춰 목록이 실시간으로 필터링되는 검색 기능을 장착했습니다.
* **재고 퀵 조절 핫버튼(Quick Adjuster) 도입**:
  * 상품 정보 수정 팝업창에서 키보드를 띄우지 않고도 **`-`**, **`+`**, **`+10`** 버튼을 터치하여 재고 수량을 1초 만에 수정 및 저장할 수 있습니다. (`StatefulBuilder` 적용으로 화면 반응형 갱신)

## 📌 6. 이커머스용 고유 주문 식별 번호(`order_code`) 파이프라인

* **주문 번호 암호화 및 체계화**:
  * 오름차순 정수(예: `1`, `2`, `3`)로 노출되던 주문 번호를 실제 전문 쇼핑몰 수준으로 암호화하여 노출합니다.
  * **포맷**: `P[주문년월일]-[패딩된 4자리 ID]-[보안해시 4글자]` (예: `P20260710-0015-E8A7`)
  * **설계 강점**: DB 마이그레이션 없이 가공하여 가볍고, 타인이 주문번호 유추를 통해 서비스의 매출액이나 주문량을 어뷰징 추적하는 보안 위험을 원천 차단합니다.
* **주문번호 텍스트 오버플로우 버그 해결**:
  * 주문번호 길이가 길어짐에 따라 좁은 화면에서 우측 텍스트 경고창(오버플로우)이 발생하던 UI 문제를 **세로 2행 복합 레이아웃(Column) 구조**로 헤더를 전면 개편하여 완벽하게 해결했습니다. (구매자용 `orders_screen.dart` 및 업주용 `seller_dashboard_screen.dart` 둘 다 적용 완료)

## 📌 7. 출시 전 보안 하드닝 (스토어 검수 대응)

출시 가정 하에 백엔드·클라이언트를 감사하고 심각도별로 조치했습니다.

* **구글 로그인 인증 우회 차단**:
  * 기존엔 `PETFIT_GOOGLE_CLIENT_ID` 미설정 시 클라이언트 `idToken`을 검증 없이 그대로 `google_id`로 써서, 아무 문자열로나 임의 계정 로그인이 가능했습니다(계정 탈취).
  * 운영 오설정(client_id 없고 dev-login도 꺼짐) 시 **503 거부**로 막고, 개발 폴백은 `dev:` 네임스페이스로 분리해 검증된 구글 계정 가로채기를 원천 차단했습니다.
* **비밀번호 해시 bcrypt 전환**:
  * `SHA-256 + 소스 하드코딩 salt`(취약) → **bcrypt**. 기존 가입자는 로그인 성공 시점에 무중단 자동 재해시로 마이그레이션됩니다.
* **인증 엔드포인트 레이트리밋**:
  * `app/ratelimit.py` 신설. 로그인·회원가입·구글·dev-login에 IP 분당 상한(기본 10회, `PETFIT_AUTH_RATE_PER_MIN`)을 걸어 무차별 대입을 방어합니다.
* **pet_id 소유권 검증(IDOR 차단)**:
  * `/tryon`·`/tryon/fourcut`에서 요청의 `pet_id`가 본인 소유인지 확인합니다(타인 펫 프로필·향후 펫 전용 LoRA 도용 방지).
* **이미지 업로드 검증**:
  * `app/uploads.py` 신설. 펫/리뷰/상품/피팅 업로드에 용량(10MB)·실제 이미지 여부(PIL 디코딩)·포맷(jpg/png/webp)을 검증하고, 확장자·MIME을 클라이언트 값 대신 **실제 디코딩된 포맷**으로 결정합니다.
* **CORS 축소**: 임의의 타 Vercel 앱까지 자격증명 요청을 허용하던 `*.vercel.app` 와일드카드를 제거했습니다.
* **DB 유실 사고 방지**: Railway 환경에서 SQLite 폴백이 감지되면 조용히 뜨는 대신 **기동을 실패**시켜(데이터 유실 전에) 알립니다.
* **로그인 세션 영속화**: Flutter 토큰을 메모리 대신 `flutter_secure_storage`에 저장해 앱 재시작에도 로그인이 유지됩니다.
* **결제 우회 경로 제거**: 결제 없이 주문완료+재고차감+AI보너스를 만들 수 있던 (구) `POST /me/orders`를 삭제했습니다. 결제는 `pending → 토스 → confirm` 단일 경로만 남았고, 재고·보너스 이중 지급이 차단됩니다. (겸사겸사 `confirm_payment`의 중첩 세션 데드락 버그도 해결.)
* **정보 노출 축소**: `/health`에서 프로바이더·DB·스토리지 구성 노출을 제거(상태만 반환), 토스/구글/생성 실패 시 내부 예외 원문을 서버 로그로만 남기고 사용자에겐 일반 메시지를 반환합니다. (우리가 던지는 재고 부족·중복 이메일 등 유저용 메시지는 유지.)

> 참고: 토스페이먼츠 실결제 연동과 dev-login 활성화는 **의도된 미완료 상태**입니다(사업자등록 후 API 키 주입 시 즉시 작동, dev-login은 개발 중 편의).
>
> 검토 후 현 설계 유지(변경 불필요): ① 생성 결과 R2 공개 URL — 키가 128비트 난수(uuid4)라 열거 불가능하고, SNS 공유·피팅 라이브러리가 공개 URL을 필요로 하므로 서명 URL 전환은 오히려 기능을 깨뜨림. ② JOBS·쿼터·레이트리밋 인메모리 — 단일 인스턴스에선 문제없음. 다중 인스턴스 확장 시 Redis 등으로 이전(출시 단계에선 과투자).
